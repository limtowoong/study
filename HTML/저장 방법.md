## 1. 서버 측 데이터베이스 (MySQL, PostgreSQL 등)  
용도: 회원가입, 로그인, 게시판 등 중요한 사용자 정보를 안전하게 저장.  
보안: 비밀번호는 반드시 해시하여 저장하며, HTTPS를 통해 데이터 전송 시 보안을 강화합니다.  
<br>

## 2. 로컬 스토리지 (Local Storage)  
용도: 사용자 설정, 테마, 마지막 방문 페이지 등 비민감한 정보를 클라이언트 측에 영구적으로 저장.  
장점: 브라우저를 닫아도 데이터가 유지되며, 서버 부하를 줄일 수 있습니다.  
주의: 민감한 정보(비밀번호 등)는 절대 저장하지 않습니다.  
<br>

## 3. 세션 스토리지 (Session Storage)  
용도: 로그인 상태, 장바구니, 일시적 정보 저장 등.  
장점: 탭이나 창을 닫으면 자동 삭제되므로 짧은 생명 주기를 가진 데이터에 적합합니다.  
주의: 다른 탭/창과 데이터가 공유되지 않으므로, 세션 유지가 필요한 경우 서버 세션과 함께 사용해야 합니다.  
<br>

## 4. 쿠키 (Cookies)  
용도: 로그인 유지, 광고 추적, 사이트 방문 기록 등 사용자 정보를 클라이언트에 저장.  
장점: 만료 시간을 설정할 수 있어 자동 로그인 등 지속적인 상태 유지에 적합.  
주의: 크기가 작고 전송 시마다 서버로 함께 전송되므로 보안에 주의해야 합니다 (ex. `HttpOnly`, `Secure` 속성 사용).  
<br>

## 5. 서버 세션 (Server-side Session)  
용도: 로그인 인증, 사용자 상태 저장 등 민감한 데이터를 안전하게 서버에 저장.  
장점: 클라이언트에 민감한 데이터를 노출하지 않음.  
주의: 서버 메모리를 사용하므로 사용자가 많을 경우 스케일링 전략이 필요합니다.  
<br>

## 6. 클라우드 스토리지 (Firebase, AWS Amplify 등)  
용도: 모바일/SPA 앱에서 사용자 데이터, 이미지, JSON 파일 등을 클라우드에 저장.  
장점: 백엔드 없이도 빠르게 앱 개발이 가능하며, 실시간 데이터 동기화 기능 제공.  
주의: 인증 및 접근 제어를 반드시 설정해야 보안 위험을 줄일 수 있습니다.  
<br>
